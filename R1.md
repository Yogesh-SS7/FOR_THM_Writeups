**TryHackMe – Containment Writeup**
Difficulty: Medium
Platform: [TryHackMe]
Operating System: Kali Linux

**Room Overview**
This room focused on incident response and AI-assisted investigation techniques.
The objective was to investigate a compromised environment, identify indicators of compromise, analyze phishing activity, inspect packet captures, and recover the final flag using integrated AI tools.
The room demonstrated how AI can accelerate repetitive investigation tasks during cybersecurity operations.

**Initial Access**
As instructed by the room, both the AttackBox and victim machine were started.
SSH access to the target machine was established using:
```bash
ssh o.deer@MACHINE_IP
```
The provided password was then entered to access the system.
At the same time, Firefox was used to access the AI assistant interface:
```text
http://MACHINE_IP:7860/?__theme=light
```
This provided access to the AI investigation environment.

---

**Initial Investigation**
The first prompt submitted to the AI assistant was:
```text
What are your capabilities and how do you think we should start this investigation?
```
While waiting for the response, I began manually exploring the environment through SSH.
Basic reconnaissance commands used:
```bash
ls
cd
pwd
```
During enumeration, a file named `pawned.txt` was discovered inside the Desktop directory.
The contents were reviewed using:
```bash
cat pawned.txt
```
This provided initial clues indicating that the system had been compromised.

---
**Mail Analysis**
Based on the clues discovered, I navigated to the Mail directory:
```bash
cd ..
cd Mail
```
The available mail files were listed using:
```bash
ls
```
To accelerate the investigation, the following prompt was submitted to the AI assistant:
```text
Can you search the files in the directory /home/o.deer/Mail for signs of phishing email using the phishing_email_detector tool and report to me what you find?
```
The AI identified a suspicious email that appeared to be related to the compromise.
The flagged mail was then manually reviewed using:
```bash
cat susmail.txt
```
(Note: filename altred for demonstration purposes.)
The email revealed characteristics commonly associated with phishing attacks, including:
urgency-based social engineering
insider-related messaging
deceptive attachment delivery

---
**Payload Investigation**
Following the lead from the phishing email, I navigated to the Downloads directory and identified the file:
```text
invoice_payload.src
```
The file contents were reviewed to understand the attack pattern and payload behavior.
At this stage, it became clear that the compromise originated from a cleverly disguised phishing campaign that relied heavily on urgency and trust manipulation.

---
**PCAP Investigation**
Further investigation was conducted inside:
```text
/home/o.deer/Documents/pcap_dumps/2025-06-17
```
To narrow the investigation timeline, I asked the AI assistant to identify the date the phishing email was received.
Each PCAP-related file was manually reviewed using `cat` to identify the file with the largest amount of activity.
Example approach used:
```bash
cat filename
```
In hindsight, a faster method would have been:
```bash
wc -c *
```
This would quickly identify the largest files by character count.
After identifying the most relevant capture file, the following prompt was submitted:
```text
Can you summarize the file in /home/o.deer/path_to_file using the pcap_file_reassembler tool and report your findings?
```
The AI-generated analysis revealed additional indicators of compromise and provided the password required to unzip the `westtech_projects` archive.

---
**Final Flag Recovery**
After extracting the archive, I navigated to:
```text
/home/o.deer/westtech_projects/thm_flags_guide.txt
```
The instructions were reviewed using:
```bash
cat thm_flags_guide.txt
```
The final step required analyzing another file using the AI assistant.
Prompt used:
```text
Use the liberty_prime tool on the file /home/o.deer/westtech_projects/thm_flags.txt and find me the flag. Note that it will contain exactly 3 prime numbers.
```
The AI assistant successfully identified and returned the final flag.

---
**Lessons Learned**
This room was an excellent demonstration of how AI-assisted tooling can significantly improve incident response workflows.
Key takeaways:
AI can accelerate phishing analysis and log investigation.
Combining manual investigation with AI-assisted tooling improves efficiency.
PCAP and email analysis remain critical skills during incident response.
Social engineering attacks often rely more on psychological manipulation than technical sophistication.

This room also inspired ideas for improving my own project, Cyber Sage, particularly by implementing AI-assisted email analysis and automated investigation tooling.
